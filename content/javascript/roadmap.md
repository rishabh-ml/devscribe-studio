# The Complete JavaScript Mastery Roadmap: Zero to Absolute Expert

This is the most comprehensive JavaScript learning roadmap available — **10 phases, 60+ topic clusters, and 200+ individual concepts** organized in precise learning order from foundational syntax through engine internals and metaprogramming. Every concept is explicitly listed so nothing is left to guesswork. The roadmap reflects the JavaScript ecosystem as of early 2026, prioritizing modern ECMAScript features (through ES2025), current tooling (Vite, Vitest, Biome), and resources updated for 2024–2025. Following this roadmap from start to finish will build complete JavaScript mastery — the kind where you understand not just *what* works but *why* it works at every level of the language.

The estimated total time is **9–14 months** of focused study at 15–20 hours per week, though individual pace varies. Each phase builds directly on the previous one; resist the urge to skip ahead.

## Phase 1: Language foundations and core syntax (Weeks 1–3)

Since you already know variables, data types, and `console.log`, this phase fills every remaining gap in your fundamental understanding before moving forward. Do not rush this — shaky foundations create confusion later.

**Variables and declarations.** Master the differences between `var`, `let`, and `const` — not just syntax but behavior. Understand that `var` is function-scoped and hoisted (initialized as `undefined`), while `let` and `const` are block-scoped and sit in the **temporal dead zone** until their declaration is reached. Know that `const` prevents reassignment but does not make objects immutable. Learn variable naming conventions (camelCase, UPPER_SNAKE_CASE for constants) and the `"use strict"` directive that catches common mistakes.

**Data types in depth.** JavaScript has **7 primitives**: `string`, `number`, `boolean`, `null`, `undefined`, `Symbol`, and `BigInt`. Everything else is a reference type (objects, arrays, functions). Understand that primitives are immutable and compared by value, while objects are compared by reference. Master the `typeof` operator and its quirks (`typeof null === "object"` is a famous bug). Learn `BigInt` for integers beyond `Number.MAX_SAFE_INTEGER` (2^53 - 1).

**Type conversion and coercion.** Explicit conversion uses `Number()`, `String()`, `Boolean()`, `parseInt()`, `parseFloat()`. Implicit coercion happens in operations like `"5" + 3` (string concatenation) versus `"5" - 3` (numeric subtraction). Memorize truthy and falsy values: `false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, and `NaN` are falsy — everything else is truthy. Understand `==` (loose equality with coercion) versus `===` (strict equality, no coercion) versus `Object.is()` (handles `NaN` and `-0` edge cases).

**Operators.** Cover every category: arithmetic (`+`, `-`, `*`, `/`, `%`, `**`), assignment (`=`, `+=`, `-=`, `*=`, `/=`, `%=`, `**=`, `&&=`, `||=`, `??=`), comparison (`==`, `===`, `!=`, `!==`, `>`, `<`, `>=`, `<=`), logical (`&&`, `||`, `!`), bitwise (`&`, `|`, `^`, `~`, `<<`, `>>`, `>>>`), unary (`typeof`, `delete`, `void`, `+`, `-`, `!`, `~`, `++`, `--`), the comma operator, the spread operator (`...`), optional chaining (`?.`), and nullish coalescing (`??`). Learn **operator precedence** — multiplication before addition, logical AND before OR, and so on.

**Expressions versus statements.** An expression produces a value (`2 + 3`, `x > 5 ? "yes" : "no"`); a statement performs an action (`if`, `for`, `let x = 5`). This distinction matters for arrow functions, ternary operators, and template literals.

**Numbers and Math.** Understand IEEE 754 floating-point precision issues (`0.1 + 0.2 !== 0.3`). Learn `NaN` (not equal to itself — use `Number.isNaN()`), `Infinity`, `-Infinity`, `Number.isFinite()`, `Number.isInteger()`. Master the `Math` object: `Math.floor()`, `Math.ceil()`, `Math.round()`, `Math.trunc()`, `Math.random()`, `Math.max()`, `Math.min()`, `Math.abs()`, `Math.pow()`, `Math.sqrt()`. Know `toFixed()` and `toPrecision()` for formatting.

**Basic I/O.** Browser: `alert()`, `prompt()`, `confirm()`. Console: `console.log()`, `console.error()`, `console.warn()`, `console.table()`, `console.dir()`, `console.time()`/`console.timeEnd()`, `console.group()`/`console.groupEnd()`, `console.assert()`, `console.trace()`.

## Phase 2: Control flow, strings, and basic data structures (Weeks 3–5)

**Conditionals.** `if`/`else if`/`else` for branching logic. `switch` statements with `case`, `default`, and fall-through behavior (always use `break` unless intentional). The ternary operator `condition ? valueIfTrue : valueIfFalse` as an expression (can be nested, but don't — it hurts readability). Short-circuit evaluation: `&&` returns the first falsy value or the last value; `||` returns the first truthy value or the last value.

**Loops.** `for` loops (initialization, condition, increment). `while` loops (condition checked before each iteration). `do...while` (executes at least once, condition checked after). `for...in` iterates over an object's enumerable string property keys (including inherited ones — use `hasOwnProperty()` to filter). `for...of` iterates over iterable values (arrays, strings, Maps, Sets, generators). Labeled statements (`outer: for (...)`) with `break outer` and `continue outer` for nested loop control. Understand **when to use which loop**: `for...of` for arrays and iterables, `for...in` for object keys, `forEach` for array side effects, and classic `for` when you need the index or early termination.

**Strings deep dive.** String creation with single quotes, double quotes, and backticks. **Template literals** with `${expression}` interpolation and multiline support. String immutability — methods return new strings. Comprehensive string methods: `length`, `charAt()`, `charCodeAt()`, `codePointAt()`, `indexOf()`, `lastIndexOf()`, `includes()`, `startsWith()`, `endsWith()`, `slice()`, `substring()` (no `substr` — it's deprecated), `toUpperCase()`, `toLowerCase()`, `trim()`, `trimStart()`, `trimEnd()`, `padStart()`, `padEnd()`, `repeat()`, `replace()`, `replaceAll()`, `split()`, `concat()`, `search()`, `match()`, `matchAll()`, `at()`, `normalize()`, `localeCompare()`. Learn **tagged template literals** — functions that process template strings (used in libraries like `styled-components` and `lit-html`).

**Arrays foundations.** Creation: literal `[]`, `new Array()`, `Array.of()`, `Array.from()` (converts iterables/array-likes to arrays). Access and modification with bracket notation. `length` property (writable — setting it shorter truncates the array). Destructuring: `const [first, second, ...rest] = array`. Spread: `[...arr1, ...arr2]` for copying and merging. Core mutating methods: `push()`, `pop()`, `shift()`, `unshift()`, `splice()`, `sort()`, `reverse()`, `fill()`, `copyWithin()`. Core non-mutating methods: `slice()`, `concat()`, `join()`, `flat()`, `flatMap()`, `includes()`, `indexOf()`, `lastIndexOf()`, `at()`. ES2023 "change by copy" methods: `toReversed()`, `toSorted()`, `toSpliced()`, `with()` — these return new arrays, leaving the original unchanged.

**Array iteration methods** (learn all of these — they are the backbone of JavaScript data manipulation): `forEach()` (side effects, no return value), `map()` (transforms each element, returns new array), `filter()` (keeps elements passing a test), `reduce()` (accumulates to a single value — master this one especially), `reduceRight()`, `find()` (first match), `findIndex()` (index of first match), `findLast()`, `findLastIndex()`, `some()` (at least one passes), `every()` (all pass), `Array.isArray()` for type checking. Learn to **chain methods**: `arr.filter(...).map(...).reduce(...)`.

**Objects foundations.** Object literal syntax `{ key: value }`. Dot notation vs bracket notation (bracket allows computed keys and special characters). Property shorthand (`{ name, age }` when variable matches key). Computed property names (`{ [dynamicKey]: value }`). Methods in objects (`{ greet() { } }`). Object destructuring: `const { name, age, address: { city } } = person` (including renaming, defaults, and nested destructuring). Spread with objects: `{ ...obj1, ...obj2 }`. Key `Object` static methods: `Object.keys()`, `Object.values()`, `Object.entries()`, `Object.assign()`, `Object.freeze()`, `Object.seal()`, `Object.fromEntries()`, `Object.hasOwn()` (ES2022, replaces `hasOwnProperty`). Understand shallow vs deep copying (`structuredClone()` for deep copies, available since 2022).

**JSON.** `JSON.stringify()` (object to string, with optional replacer and space arguments) and `JSON.parse()` (string to object, with optional reviver). JSON format rules (double quotes, no trailing commas, no functions, no `undefined`). The `toJSON()` method for custom serialization.

## Phase 3: Functions, scope, and closures (Weeks 5–8)

This phase covers the concepts that separate beginners from intermediate developers. Take your time — closures and scope are the intellectual foundation of the entire language.

**Function forms.** Function declarations are hoisted (can be called before they appear in code). Function expressions are assigned to variables and are **not hoisted**. Arrow functions (`=>`) have shorter syntax, implicit return for single expressions, no own `this` binding, no `arguments` object, cannot be used as constructors. Immediately Invoked Function Expressions (IIFEs): `(function() { ... })()` — historically used for scope isolation, now largely replaced by block scoping with `let`/`const` and ES modules. The `Function` constructor `new Function('a', 'b', 'return a + b')` creates functions from strings — understand it but avoid it in production.

**Parameters and arguments.** Default parameters (`function greet(name = "World")`). Rest parameters (`function sum(...nums)`) collect remaining arguments into an array. The legacy `arguments` object (array-like, not available in arrow functions). Parameter destructuring (`function draw({ x = 0, y = 0, color = "black" })`).

**Scope.** **Global scope** — variables accessible everywhere (avoid polluting it). **Function scope** — `var` declarations are scoped to the enclosing function. **Block scope** — `let` and `const` are scoped to the enclosing `{ }` block. **Lexical scope** — a function's scope is determined by where it is *written*, not where it is *called*. The **scope chain** — when a variable is referenced, JavaScript looks in the current scope, then the outer scope, up to the global scope. **Module scope** — top-level variables in ES modules are scoped to that module, not global.

**Hoisting rules.** `var` declarations are hoisted and initialized to `undefined`. `let` and `const` declarations are hoisted but not initialized — accessing them before declaration throws a `ReferenceError` (the **temporal dead zone**). Function declarations are fully hoisted (both name and body). Function expressions and arrow functions follow the hoisting rules of their variable (`var`/`let`/`const`). Class declarations are hoisted but not initialized (temporal dead zone applies).

**Closures.** A closure is a function that retains access to its lexical scope even when executed outside that scope. This is not a special feature — it is a natural consequence of lexical scoping and first-class functions. Every function in JavaScript forms a closure. Practical applications: data privacy (encapsulating state), factory functions, partial application, event handlers that remember state, module patterns. Classic pitfall: closures in loops with `var` (all iterations share the same variable — fix with `let` or an IIFE).

**Higher-order functions.** Functions that take functions as arguments or return functions. `map`, `filter`, `reduce` are higher-order functions. **Callback functions** are functions passed as arguments to be called later — the foundation of async JavaScript. Understand **callback hell** (deeply nested callbacks) as motivation for Promises.

**Recursion.** A function calling itself with a base case to stop. Understand the **call stack** — each recursive call adds a frame. Know the risk of stack overflow with deep recursion. Tail call optimization (specified in ES2015 but only implemented in Safari). Common recursive patterns: factorial, Fibonacci, tree traversal, deep object cloning, nested data flattening.

**Function composition and functional patterns.** `compose(f, g)(x) = f(g(x))` and `pipe(f, g)(x) = g(f(x))`. **Currying** transforms `f(a, b, c)` into `f(a)(b)(c)` — each call returns a new function until all arguments are supplied. **Partial application** fixes some arguments, returning a function that takes the rest (achievable with `bind()` or closures). **Memoization** caches function results for identical inputs — useful for expensive computations.

## Phase 4: The `this` keyword, prototypes, and OOP (Weeks 8–11)

**The `this` keyword — every rule.** `this` is not static; it depends on *how* a function is called. In the **global context**, `this` is `window` (browser) or `globalThis`. In a regular function (non-strict), `this` is `globalThis`; in strict mode, `this` is `undefined`. In an **object method**, `this` is the object before the dot. In a **constructor** (called with `new`), `this` is the newly created object. In an **arrow function**, `this` is inherited from the enclosing lexical scope — arrow functions do not have their own `this`. In an **event handler**, `this` is the element the listener is attached to (unless using an arrow function). **Explicit binding**: `call(thisArg, arg1, arg2)` and `apply(thisArg, [args])` invoke immediately with a specified `this`; `bind(thisArg)` returns a new function permanently bound to `thisArg`. In **classes**, `this` refers to the instance; methods can lose their `this` when extracted — bind them in the constructor or use class fields with arrow functions.

**Prototypes and the prototype chain.** Every object has an internal `[[Prototype]]` link (accessed via `Object.getPrototypeOf()` or the deprecated `__proto__`). When a property is accessed, JavaScript walks up the prototype chain until it finds the property or reaches `null`. Constructor functions have a `.prototype` property that becomes the `[[Prototype]]` of instances created with `new`. `Object.create(proto)` creates an object with a specified prototype. `instanceof` checks the prototype chain. **Prototypal inheritance** means objects inherit directly from other objects — unlike class-based languages where instances inherit from class blueprints.

**Constructor functions.** The `new` keyword: (1) creates an empty object, (2) sets its prototype to the constructor's `.prototype`, (3) binds `this` to the new object, (4) executes the constructor, (5) returns the object (unless the constructor explicitly returns a different object). `hasOwnProperty()` and `Object.hasOwn()` distinguish own properties from inherited ones.

**ES6+ classes.** Class declarations and expressions. The `constructor` method. Instance methods and **static methods** (`static myMethod()`). **Static properties** (`static count = 0`). **Public class fields** (`name = "default"`) — declared outside the constructor. **Private fields and methods** (`#privateField`, `#privateMethod()`) — truly private, enforced by the engine. **Getters and setters** (`get fullName()`, `set fullName(value)`). Inheritance with `extends` — the child class inherits all parent methods and properties. `super()` in the constructor calls the parent constructor; `super.method()` calls a parent method. Static methods are also inherited. The `instanceof` operator and `Symbol.hasInstance` for custom behavior. **Mixins** — since JavaScript supports single inheritance, mixins are objects whose methods are copied onto a class prototype to simulate multiple inheritance. Abstract class patterns — base classes that throw errors if instantiated directly.

**OOP principles in JavaScript.** **Encapsulation** — hiding internal state with private fields, exposing controlled interfaces through methods. **Abstraction** — simplifying complex systems by exposing only necessary interfaces. **Polymorphism** — different classes responding to the same method name in different ways (method overriding). **Inheritance** — sharing behavior through prototype chains or `extends`. **SOLID principles** applied to JS: Single Responsibility (one class/module per concern), Open/Closed (extend behavior without modifying source), Liskov Substitution (subtypes must be substitutable for parent types), Interface Segregation (prefer specific interfaces over general ones), Dependency Inversion (depend on abstractions, not concretions). **Composition over inheritance** — favor composing objects from small, focused pieces over deep inheritance hierarchies.

## Phase 5: Asynchronous JavaScript and error handling (Weeks 11–14)

Async JavaScript is where most learners stumble. This phase builds understanding from the ground up — starting with how JavaScript executes code, then callbacks, then Promises, then async/await.

**The event loop in detail.** JavaScript is **single-threaded** — it has one call stack. The event loop manages concurrency by coordinating between the call stack, the **Web APIs** (setTimeout, fetch, DOM events — handled by the browser/Node.js), the **macrotask queue** (setTimeout callbacks, setInterval callbacks, I/O operations, UI rendering), and the **microtask queue** (Promise `.then`/`.catch`/`.finally` callbacks, `queueMicrotask()`, `MutationObserver` callbacks). Execution order: (1) execute all synchronous code on the call stack, (2) drain the entire microtask queue, (3) execute one macrotask, (4) drain microtasks again, (5) optionally render, (6) repeat. This means Promises always resolve before setTimeout, even `setTimeout(fn, 0)`. Node.js has a slightly different event loop with additional phases (timers, pending callbacks, poll, check, close callbacks) and `process.nextTick()` (runs before other microtasks).

**Callbacks.** Functions passed as arguments to be called when an operation completes. Node.js convention: `callback(error, result)`. **Callback hell** (pyramid of doom) occurs when nested callbacks create deeply indented, hard-to-read code.

**Promises.** A Promise is an object representing the eventual completion or failure of an async operation. States: **pending**, **fulfilled**, **rejected**. Create with `new Promise((resolve, reject) => { ... })`. Chain with `.then(onFulfilled, onRejected)`, handle errors with `.catch(handler)`, run cleanup with `.finally(handler)`. **Promise chaining** — each `.then()` returns a new Promise, enabling sequential async operations. Static methods: `Promise.all(iterable)` waits for all to fulfill (rejects on first rejection), `Promise.allSettled(iterable)` waits for all to settle regardless of outcome, `Promise.race(iterable)` resolves/rejects with the first to settle, `Promise.any(iterable)` resolves with the first to fulfill (rejects only if all reject with `AggregateError`), `Promise.resolve(value)` and `Promise.reject(reason)` create immediately settled Promises. **ES2024**: `Promise.withResolvers()` returns `{ promise, resolve, reject }` in one call — eliminates the awkward pattern of extracting resolve/reject from the constructor. **ES2025**: `Promise.try(fn)` wraps sync-or-async function calls in a Promise for consistent error handling.

**Async/await.** `async` before a function makes it return a Promise. `await` pauses execution inside an `async` function until the Promise settles. Error handling with `try`/`catch` around `await` calls. Running async operations in parallel: `const [a, b] = await Promise.all([fetchA(), fetchB()])`. **Top-level await** (ES2022) — use `await` at the module's top level in ES modules. Common pitfalls: unnecessary sequential awaits when operations could run in parallel; forgetting error handling.

**AbortController.** `const controller = new AbortController()` creates a controller with a `.signal` property. Pass `signal` to `fetch()` options to make requests cancellable. Call `controller.abort()` to cancel. Essential for cleanup in React effects, timeouts, and user-initiated cancellations.

**Scheduling.** `setTimeout(fn, delay)` and `setInterval(fn, interval)`. `clearTimeout()` and `clearInterval()`. `requestAnimationFrame(callback)` for smooth visual updates synchronized with the display refresh rate. `requestIdleCallback(callback)` for deferring non-critical work. `queueMicrotask(fn)` for scheduling a microtask explicitly.

**Error handling.** `try`/`catch`/`finally` blocks. The `throw` statement (can throw any value, but prefer `Error` objects). Built-in error types: `Error`, `TypeError`, `ReferenceError`, `SyntaxError`, `RangeError`, `URIError`, `EvalError`, `AggregateError` (ES2021). **Custom error classes**: `class ValidationError extends Error { constructor(message) { super(message); this.name = "ValidationError"; } }`. The `error.cause` property (ES2022) for wrapping errors: `throw new Error("Failed", { cause: originalError })`. **Stack traces** — read them bottom-to-top; the top frame is where the error occurred. Error handling in async code: use `try`/`catch` with `await`, or `.catch()` on Promises. **Unhandled rejections**: listen for `unhandledrejection` events in the browser, `process.on('unhandledRejection')` in Node.js. Debugging strategies: breakpoints (conditional, DOM, XHR), the `debugger` statement, Chrome DevTools Sources panel, `console.trace()` for stack inspection.

## Phase 6: DOM, events, browser APIs, and network requests (Weeks 14–18)

This phase transitions from pure language knowledge to real-world browser programming. You'll learn to manipulate web pages, handle user interaction, store data, and communicate with servers.

**DOM tree and traversal.** The DOM represents HTML as a tree of nodes. Node types: element nodes, text nodes, comment nodes. Traversal properties: `parentNode`, `parentElement`, `childNodes` (NodeList, includes text), `children` (HTMLCollection, elements only), `firstChild`/`firstElementChild`, `lastChild`/`lastElementChild`, `nextSibling`/`nextElementSibling`, `previousSibling`/`previousElementSibling`.

**Selecting elements.** `document.getElementById()`, `document.getElementsByClassName()` (live HTMLCollection), `document.getElementsByTagName()` (live), `document.querySelector()` (first match, CSS selector), `document.querySelectorAll()` (static NodeList). Prefer `querySelector`/`querySelectorAll` for flexibility. `element.closest(selector)` walks up the tree to find the nearest matching ancestor. `element.matches(selector)` checks if an element matches a selector.

**Creating and modifying elements.** `document.createElement()`, `document.createTextNode()`, `document.createDocumentFragment()`. Inserting: `parent.appendChild(child)`, `parent.insertBefore(new, reference)`, `element.append()` / `element.prepend()` (accept multiple nodes and strings), `element.after()` / `element.before()`, `element.replaceWith()`, `parent.replaceChild(new, old)`. Removing: `element.remove()`, `parent.removeChild(child)`. Cloning: `element.cloneNode(deep)`. **innerHTML** vs **textContent** — `innerHTML` parses HTML (XSS risk if inserting user input), `textContent` sets raw text safely. `outerHTML` replaces the element entirely.

**Attributes, classes, and styles.** `element.getAttribute()`, `element.setAttribute()`, `element.removeAttribute()`, `element.hasAttribute()`. **`dataset`** for `data-*` attributes (`element.dataset.userId`). Class manipulation: `element.classList.add()`, `.remove()`, `.toggle()`, `.contains()`, `.replace()`. Inline styles: `element.style.propertyName` (camelCase). Computed styles: `getComputedStyle(element)`. Element geometry: `offsetWidth`/`offsetHeight` (including border), `clientWidth`/`clientHeight` (padding box), `scrollWidth`/`scrollHeight`, `getBoundingClientRect()` (position relative to viewport).

**Events.** `element.addEventListener(type, handler, options)` where options can include `capture`, `once`, `passive`, `signal` (for AbortController removal). `element.removeEventListener()`. The **event object**: `event.type`, `event.target` (element that triggered the event), `event.currentTarget` (element the handler is attached to), `event.preventDefault()` (stop default browser behavior), `event.stopPropagation()` (stop bubbling/capturing). **Event phases**: capturing (top-down), target, bubbling (bottom-up). **Event delegation** — attach a single listener to a parent and use `event.target` to identify which child was clicked; dramatically reduces the number of listeners. **Custom events**: `new CustomEvent('myEvent', { detail: { data }, bubbles: true })` dispatched with `element.dispatchEvent()`. Key event types: mouse events (`click`, `dblclick`, `mousedown`, `mouseup`, `mousemove`, `mouseenter`, `mouseleave`, `mouseover`, `mouseout`, `contextmenu`), keyboard events (`keydown`, `keyup` — use `event.key` and `event.code`), form events (`submit`, `change`, `input`, `focus`, `blur`, `reset`), pointer events (unified mouse/touch/stylus), scroll events, drag-and-drop events, touch events, clipboard events (`copy`, `cut`, `paste`), page lifecycle events (`DOMContentLoaded`, `load`, `beforeunload`, `unload`).

**Advanced DOM observers.** **MutationObserver** watches for DOM changes (child additions/removals, attribute changes, text content changes). **IntersectionObserver** detects when elements enter or leave the viewport — essential for lazy loading images, infinite scroll, and scroll-triggered animations. **ResizeObserver** fires when an element's dimensions change.

**Browser data storage.** **localStorage** persists across sessions (same origin), **sessionStorage** clears when the tab closes. Both use `getItem()`, `setItem()`, `removeItem()`, `clear()` — store strings only (use `JSON.stringify`/`JSON.parse` for objects). **5–10MB limit**. **Cookies** — `document.cookie` for client-side access; attributes include `expires`, `max-age`, `path`, `domain`, `secure`, `SameSite`, `HttpOnly` (server-set only). **4KB limit** per cookie, sent with every HTTP request. **IndexedDB** — a full client-side database for large amounts of structured data, supporting transactions, indexes, and cursors.

**Fetch API and network requests.** `fetch(url, options)` returns a Promise that resolves to a `Response` object. `response.json()`, `response.text()`, `response.blob()`, `response.arrayBuffer()`. Request options: `method`, `headers` (Headers object or plain object), `body` (string, FormData, Blob, URLSearchParams), `mode` ('cors', 'no-cors', 'same-origin'), `credentials` ('same-origin', 'include', 'omit'), `signal` (AbortController). HTTP methods: GET, POST, PUT, PATCH, DELETE. Status codes: 2xx (success), 3xx (redirect), 4xx (client error), 5xx (server error). Fetch does **not** reject on HTTP errors — check `response.ok` or `response.status`. `FormData` for multipart form submissions. `URL` and `URLSearchParams` for URL construction. **CORS** (Cross-Origin Resource Sharing) — understand preflight requests (OPTIONS), the `Access-Control-Allow-Origin` header, and why CORS errors happen. Legacy: `XMLHttpRequest` — understand it exists but use `fetch` for new code.

**Other browser APIs.** **History API** (`pushState`, `replaceState`, `popstate` event) for SPA routing. **Geolocation API** (`navigator.geolocation.getCurrentPosition()`). **Notifications API** (`Notification.requestPermission()`, `new Notification()`). **Clipboard API** (`navigator.clipboard.readText()`, `.writeText()`). **Blob and File API** for binary data handling. **Canvas API** basics (2D drawing context, `getContext('2d')`, shapes, text, images). **Web Audio API** basics (AudioContext, oscillators, gain nodes). **matchMedia()** for responsive JS (listening to media query changes). **Page Visibility API** (`document.visibilityState`). The `navigator` and `window` objects.

**Script loading.** `<script>` blocks parsing. `<script defer>` downloads in parallel, executes after parsing, maintains order. `<script async>` downloads in parallel, executes immediately when ready (no order guarantee). Dynamic script loading with `document.createElement('script')`. `<link rel="preload">` and `<link rel="prefetch">` for resource hints.

## Phase 7: Modules, modern ES features, and iteration protocols (Weeks 18–22)

**ES Modules (ESM).** `export` (named exports: `export function myFunc() {}`), `export default` (one per module), `import { myFunc } from './module.js'`, `import defaultExport from './module.js'`, `import * as module from './module.js'`. Re-exporting: `export { default } from './other.js'`. **Dynamic imports**: `const module = await import('./heavy-module.js')` — returns a Promise, enabling code splitting and lazy loading. `import.meta` provides metadata (e.g., `import.meta.url`). Module scope — top-level variables are scoped to the module. Modules are always in strict mode. Modules are evaluated only once (singleton behavior). **ES2025 Import Attributes**: `import data from './data.json' with { type: 'json' }` for native JSON module imports.

**CommonJS (Node.js).** `require()` for importing, `module.exports` or `exports.myFunc` for exporting. Synchronous loading. Understand the difference: CommonJS loads at runtime, ESM is statically analyzable (enabling tree shaking). Modern Node.js supports both — use `"type": "module"` in `package.json` for ESM.

**Maps, Sets, WeakMaps, and WeakSets.** **Map** — key-value pairs where keys can be any type (not just strings). Methods: `set()`, `get()`, `has()`, `delete()`, `clear()`, `size`. Iterable with `for...of`, `forEach()`, `keys()`, `values()`, `entries()`. **Set** — unique values. Methods: `add()`, `has()`, `delete()`, `clear()`, `size`. **ES2025 Set methods**: `union()`, `intersection()`, `difference()`, `symmetricDifference()`, `isSubsetOf()`, `isSupersetOf()`, `isDisjointFrom()` — these are a major addition. **WeakMap** — keys must be objects (or symbols), keys are weakly held (garbage-collectable). No iteration, no `size`. Use cases: private data storage, caching without memory leaks. **WeakSet** — like Set but for objects, weakly held. Use case: tracking objects without preventing garbage collection.

**Symbols.** `Symbol('description')` creates a unique identifier. `Symbol.for('key')` creates/retrieves global symbols. Use cases: unique object property keys that don't clash, defining protocols. **Well-known symbols**: `Symbol.iterator` (defines iteration protocol), `Symbol.toPrimitive` (controls type conversion), `Symbol.hasInstance` (customizes `instanceof`), `Symbol.species` (controls constructor for derived objects), `Symbol.toStringTag` (customizes `Object.prototype.toString` output), `Symbol.isConcatSpreadable`, `Symbol.asyncIterator`.

**Iterators and generators.** The **iterable protocol**: an object is iterable if it has a `[Symbol.iterator]()` method that returns an iterator. The **iterator protocol**: an iterator has a `next()` method returning `{ value, done }`. Custom iterables: implement `[Symbol.iterator]()` on any object. **Generator functions** (`function* gen() { yield 1; yield 2; }`) create iterators that can pause execution. `yield` pauses and returns a value; calling `next()` resumes. Generators are lazy — values are computed on demand. **`yield*`** delegates to another generator or iterable. Generators can receive values: `const input = yield output`. **Async generators** (`async function* gen() { yield await fetch(...) }`) combine generators with async operations. **`for await...of`** iterates over async iterables.

**ES2024 features.** `Object.groupBy(iterable, callback)` and `Map.groupBy()` for grouping data. `Promise.withResolvers()`. `ArrayBuffer.prototype.transfer()` and `.resize()` for zero-copy buffer operations. `String.prototype.isWellFormed()` and `.toWellFormed()` for UTF-16 validation. `Atomics.waitAsync()` for async shared memory synchronization. **RegExp `v` flag** (unicodeSets) enabling set operations in character classes (`[A--B]` for difference, `[A&&B]` for intersection) and Unicode string properties.

**ES2025 features.** **Iterator helpers** — `Iterator.prototype` gains `.map()`, `.filter()`, `.take()`, `.drop()`, `.flatMap()`, `.reduce()`, `.toArray()`, `.forEach()`, `.some()`, `.every()`, `.find()`, plus `Iterator.from()`. These enable **lazy, functional processing** of any iterable without intermediate arrays — a major paradigm shift. **New Set methods** (listed above). **Import Attributes** with `with { type: 'json' }`. **`RegExp.escape()`** for safe regex construction from strings. **RegExp pattern modifiers** (`(?i:...)` for inline flag toggling). **Duplicate named capture groups** across alternatives. **`Promise.try()`**. **`Float16Array`** and `Math.f16round()` for half-precision floating-point. **Explicit Resource Management** (`using` with `Symbol.dispose`) has reached Stage 4 and will be in ES2026 — learn it now as it fundamentally changes how resources (file handles, database connections, locks) are cleaned up.

**ECMAScript version history (ES2015–ES2023 quick reference).** ES2015 was the massive modernization (let/const, arrow functions, classes, Promises, modules, destructuring, template literals, Map/Set, Symbol, Proxy/Reflect, generators, iterators). ES2016 added `Array.includes()` and `**`. ES2017 brought `async`/`await`, `Object.values/entries`, string padding. ES2018 added async iteration, object spread, `Promise.finally()`, regex named groups and lookbehind. ES2019 brought `flat`/`flatMap`, `Object.fromEntries()`, optional catch binding. ES2020 added optional chaining, nullish coalescing, `BigInt`, `Promise.allSettled()`, dynamic import, `globalThis`. ES2021 introduced `String.replaceAll()`, `Promise.any()`, logical assignment (`||=`, `&&=`, `??=`), numeric separators, `WeakRef`/`FinalizationRegistry`. ES2022 brought class fields (public/private/static), top-level await, `.at()`, `Object.hasOwn()`, `error.cause`, RegExp `/d` flag. ES2023 added change-by-copy array methods (`toSorted`, `toReversed`, `toSpliced`, `with`), `findLast`/`findLastIndex`, symbols as WeakMap keys.

**Proposals to watch (Stage 3, not yet finalized).** **Temporal API** — a comprehensive date/time replacement for the flawed `Date` object, with immutable types, proper time zone support, and duration arithmetic (available via polyfill). **Decorators** — class-level metaprogramming for adding behavior declaratively (implementations in progress but no browser has shipped yet). **Pattern Matching** is still at Stage 2. **Record & Tuple** was **withdrawn** in April 2025 — do not invest time learning it; a successor "Composites" proposal is in early stages.

## Phase 8: Regex, property descriptors, Proxy/Reflect, and advanced patterns (Weeks 22–26)

**Regular expressions — complete coverage.** Syntax: `/pattern/flags`. **Flags**: `g` (global), `i` (case-insensitive), `m` (multiline — `^`/`$` match line starts/ends), `s` (dotAll — `.` matches newlines), `u` (unicode), `v` (unicodeSets, ES2024), `y` (sticky — matches at `lastIndex` position), `d` (hasIndices — provides match index info). **Character classes**: `\d` (digit), `\D`, `\w` (word char), `\W`, `\s` (whitespace), `\S`, `.` (any char). **Unicode property escapes**: `\p{Letter}`, `\p{Script=Greek}` (with `u` or `v` flag). **Anchors**: `^` (start), `$` (end), `\b` (word boundary). **Quantifiers**: `*` (0+), `+` (1+), `?` (0 or 1), `{n}`, `{n,}`, `{n,m}`. Greedy (default) vs lazy (`*?`, `+?`). **Groups**: capturing `(...)`, non-capturing `(?:...)`, **named groups** `(?<name>...)`. **Backreferences**: `\1` or `\k<name>`. **Alternation**: `|`. **Lookahead**: positive `(?=...)`, negative `(?!...)`. **Lookbehind**: positive `(?<=...)`, negative `(?<!...)`. **Methods**: `regex.test(str)`, `regex.exec(str)`, `str.match(regex)`, `str.matchAll(regex)` (returns iterator), `str.replace(regex, replacement)`, `str.replaceAll(regex, replacement)`, `str.search(regex)`, `str.split(regex)`. **ES2025**: `RegExp.escape(string)` for safe pattern construction; inline pattern modifiers `(?i:...)` for flag scoping; duplicate named capture groups. Beware **catastrophic backtracking** — nested quantifiers on overlapping patterns can freeze execution.

**Property descriptors and object configuration.** Every property has a descriptor with `value`, `writable`, `enumerable`, `configurable`. Access with `Object.getOwnPropertyDescriptor()` and `Object.getOwnPropertyDescriptors()`. Define with `Object.defineProperty(obj, prop, descriptor)` and `Object.defineProperties()`. **Getters and setters**: `get propertyName() {}` and `set propertyName(val) {}` — computed properties that look like data access. `Object.preventExtensions()` — no new properties. `Object.seal()` — no new properties, existing non-configurable. `Object.freeze()` — sealed + no value changes (shallow only). `Object.isExtensible()`, `Object.isSealed()`, `Object.isFrozen()`.

**Proxy and Reflect.** `new Proxy(target, handler)` intercepts fundamental operations. **Handler traps**: `get(target, prop, receiver)`, `set(target, prop, value, receiver)`, `has(target, prop)` (intercepts `in`), `deleteProperty(target, prop)`, `apply(target, thisArg, args)` (function calls), `construct(target, args, newTarget)` (the `new` operator), `getPrototypeOf`, `setPrototypeOf`, `isExtensible`, `preventExtensions`, `getOwnPropertyDescriptor`, `defineProperty`, `ownKeys` (intercepts `Object.keys()`, `for...in`, etc.). **Reflect** provides default implementations of each trap — always use `Reflect.get()`, `Reflect.set()`, etc. inside traps to preserve default behavior. `Proxy.revocable()` creates a proxy that can be permanently disabled. **Real-world use cases**: input validation (reject invalid property assignments), logging and debugging (trace property access), default values (return defaults for missing properties), reactive systems (Vue 3 uses Proxy for reactivity), negative array indexing, access control, immutable views.

**WeakRef and FinalizationRegistry.** `new WeakRef(targetObject)` holds a weak reference that doesn't prevent garbage collection; access with `.deref()` (returns `undefined` if collected). `new FinalizationRegistry(callback)` registers callbacks invoked when registered objects are garbage-collected. Use cases: caches that automatically release memory, observing object lifetimes. These are advanced — garbage collection timing is non-deterministic, so don't rely on them for critical logic.

**Metaprogramming.** Symbols as protocol hooks (controlling how objects behave with operators and built-in functions). Tagged templates for DSLs (domain-specific languages). `eval()` and `new Function()` — understand them but avoid in production (security risk, performance penalty, prevents optimization). Property descriptors for fine-grained object control.

## Phase 9: Design patterns, FP, testing, and memory management (Weeks 26–32)

**Design patterns in JavaScript.** **Creational**: Singleton (one instance, e.g., app configuration), Factory (creates objects without specifying exact class), Builder (step-by-step complex object construction), Constructor (the `new` keyword pattern). **Structural**: Module (encapsulation via closures or ES modules), Revealing Module (expose only chosen members), Decorator (dynamically add behavior to objects), Facade (simplify complex subsystem interfaces), Adapter (wrap an incompatible interface), Proxy (controlled access to an object). **Behavioral**: Observer/PubSub (event-driven communication — the foundation of event emitters), Mediator (centralized communication hub), Command (encapsulate actions as objects), Strategy (interchangeable algorithms), State (behavior changes based on internal state), Iterator (sequential access to collection elements). **MVC/MVVM** — Model-View-Controller and Model-View-ViewModel architectural patterns. **Anti-patterns** to avoid: God objects, callback hell, tight coupling, premature optimization.

**Functional programming in JavaScript.** **Pure functions** — same inputs always produce the same output, no side effects. **Immutability** — never mutate data; create new copies (`Object.freeze()`, spread operators, `structuredClone()`, libraries like Immer). **Referential transparency** — an expression can be replaced by its value without changing behavior. **Function composition** — combining small functions into pipelines. **Currying** and **partial application** for creating specialized functions from general ones. **Functors** — objects that implement `map()` (arrays are functors). **Monads** (basic concept) — functors that implement `flatMap()` for handling side effects, nullability (`Maybe` monad), or async operations (Promises are monads). **Transducers** — composable transformation pipelines that avoid intermediate collections, working with any data source (arrays, streams, channels). **Point-free style** — defining functions without explicitly mentioning their arguments.

**Testing JavaScript applications.** **Unit testing** — test individual functions/modules in isolation. **Integration testing** — test how modules work together. **End-to-end (E2E) testing** — simulate real user workflows through the full application. **Test-Driven Development (TDD)**: write failing test → write minimal code to pass → refactor. **Behavior-Driven Development (BDD)**: describe behavior in natural language (`describe`/`it`/`expect`).

**Testing frameworks.** **Vitest** is the recommended framework for new projects in 2025 — built on Vite, **10–20x faster than Jest** in watch mode, native ESM and TypeScript support, Jest-compatible API. Vitest 3 (early 2025) and Vitest 4 (October 2025) brought stable browser mode and visual regression testing. **Jest** remains widely used (35M weekly downloads) — mature ecosystem, great for React Native and existing codebases. Key concepts: `describe()` and `it()`/`test()` blocks, `expect()` assertions with matchers (`.toBe()`, `.toEqual()`, `.toThrow()`, `.toHaveBeenCalled()`), `beforeEach()`/`afterEach()` setup/teardown, **mocking** (`vi.fn()` in Vitest, `jest.fn()` in Jest) for isolating units, **spying** on existing functions, **code coverage** reports (`--coverage` flag). Testing async code: `await` in tests, `.resolves`/`.rejects` matchers. **Snapshot testing** for UI components. **E2E tools**: **Playwright** (by Microsoft, cross-browser, recommended for new projects) and **Cypress** (great DX, primarily Chromium).

**Memory management.** JavaScript uses automatic **garbage collection**. The primary algorithm is **mark-and-sweep**: starting from root references (global object, current call stack), the GC marks all reachable objects, then sweeps (frees) unreachable ones. The V8 engine uses **generational GC** — a fast "minor GC" for short-lived objects in the young generation, and a slower "major GC" for long-lived objects in the old generation. **Memory leaks** occur when references are unintentionally retained: forgotten timers/intervals, detached DOM nodes still referenced in JS, closures holding onto large scopes, growing arrays/maps never cleaned, event listeners never removed, global variables. **Chrome DevTools memory tools**: Memory tab for heap snapshots (compare snapshots to find leaked objects), Allocation timeline for identifying allocation patterns, Performance tab flame charts for execution profiling. WeakMap/WeakSet/WeakRef help prevent leaks by allowing garbage collection of keys/references.

**Performance optimization.** **Debouncing** — delay execution until input stops (e.g., search-as-you-type). **Throttling** — limit execution to once per interval (e.g., scroll handlers). **Lazy loading** — defer loading resources until needed (`loading="lazy"` for images, dynamic `import()` for code). **Code splitting** — break bundles into smaller chunks loaded on demand. **Tree shaking** — eliminate unused exports (requires ES modules and a bundler like Vite/Webpack). **Core Web Vitals** (2025): Largest Contentful Paint (LCP, target < 2.5s), Interaction to Next Paint (INP, replaced FID in March 2024, target < 200ms), Cumulative Layout Shift (CLS). **Performance measurement**: `performance.now()`, `performance.mark()`/`performance.measure()`, the `PerformanceObserver` API, Lighthouse audits, Chrome DevTools Performance tab. `requestAnimationFrame` for smooth 60fps animations. Web Workers for offloading CPU-intensive work off the main thread.

## Phase 10: TypeScript, Node.js, build tools, security, and Web Components (Weeks 32–40)

This final phase rounds out your JavaScript mastery with the essential surrounding ecosystem. TypeScript is no longer optional — **over 90% of JavaScript developers use it** according to recent surveys, and Node.js 22+ runs `.ts` files natively.

**TypeScript fundamentals.** TypeScript adds static types to JavaScript — code is type-checked at compile time, then the types are stripped to produce plain JavaScript. **Basic types**: `string`, `number`, `boolean`, `null`, `undefined`, `any` (opt out of type checking), `unknown` (type-safe `any` — must narrow before use), `void`, `never` (impossible values), `bigint`, `symbol`. **Arrays**: `number[]` or `Array<number>`. **Tuples**: `[string, number]`. **Enums**: `enum Direction { Up, Down, Left, Right }`. **Interfaces** define object shapes: `interface User { name: string; age: number; email?: string }`. **Type aliases**: `type ID = string | number`. **Union types** (`string | number`) and **intersection types** (`TypeA & TypeB`). **Generics** for reusable, type-safe code: `function identity<T>(arg: T): T { return arg; }`. **Type guards**: `typeof`, `instanceof`, custom type predicates (`function isString(val: any): val is string`). **Utility types**: `Partial<T>`, `Required<T>`, `Readonly<T>`, `Pick<T, K>`, `Omit<T, K>`, `Record<K, V>`, `ReturnType<T>`, `Parameters<T>`, `Awaited<T>`. **Strict mode** (`"strict": true` in tsconfig.json) enables all strict checks — always use it. **Declaration files** (`.d.ts`) provide types for JavaScript libraries. **Type inference** — TypeScript infers types where possible; you don't annotate everything. TypeScript 5.9 is current; **TypeScript 7.0 (Project Corsa)** is a complete rewrite in Go promising ~10x faster builds — expected early 2026.

**Node.js fundamentals.** Node.js is a JavaScript runtime built on Chrome's V8 engine. **Node.js 24** is the current LTS (recommended for production); **Node.js 22** is in active LTS. The `node` command runs JS/TS files. Key built-in modules: **`fs`** (file system — `readFileSync`, `writeFileSync`, `readFile`, `writeFile`, `mkdir`, `readdir`, and the promise-based `fs/promises` API), **`path`** (`join`, `resolve`, `basename`, `dirname`, `extname`), **`http`/`https`** for creating servers, **`url`** and `URLSearchParams`, **`os`** for system info, **`events`** (EventEmitter pattern), **`crypto`** for hashing and encryption, **`child_process`** (`exec`, `spawn` for running system commands), **`stream`** (Readable, Writable, Transform, Duplex — essential for processing large data), **`buffer`** (binary data handling). **Environment variables**: `process.env` and `.env` files with `dotenv` (or built-in `--env-file` flag in Node 20+). Node.js now natively supports TypeScript (unflagged since Node 22.18.0), has a built-in test runner, a Permission Model for sandboxing, and Web Storage. **npm** is the default package manager (ships with Node); understand `package.json` anatomy (`name`, `version`, `main`/`module`/`exports`, `scripts`, `dependencies`, `devDependencies`, `peerDependencies`, `engines`), **semantic versioning** (`^1.2.3` = compatible updates, `~1.2.3` = patch updates), `package-lock.json`, and `npx` for running packages without installing.

**Build tools and the modern ecosystem.** **Package managers**: **pnpm** is the 2025 recommendation (fastest, most disk-efficient, strictest dependency isolation); **npm** is the default; **Yarn Berry (v4)** is feature-rich but losing ground. **Bun** is an alternative all-in-one runtime/package-manager/bundler (acquired by Anthropic in November 2025). **Deno 2** is a security-first runtime with native TypeScript, now npm-compatible. **Bundlers**: **Vite** is the dominant build tool — uses esbuild for dev server speed and Rollup for production builds. **Webpack** is legacy but still widespread. **esbuild** and **Rollup** are often used under the hood. **Turbopack** (Rust-based, by Vercel) is built into Next.js. **Rspack** is a Webpack-compatible Rust alternative gaining enterprise adoption. **Transpilers**: **Babel** compiles modern JS to older syntax; **SWC** (Rust-based) is a faster alternative used by Next.js. **Linting**: **ESLint** with flat config (the new default) is the standard; **Biome** (Rust-based, **25x faster than Prettier, 15x faster than ESLint**) is the emerging challenger combining linting and formatting in one tool. **Formatting**: **Prettier** for opinionated code formatting, or Biome as an all-in-one solution.

**Security.** **XSS (Cross-Site Scripting)** — injecting malicious scripts through user input. Prevention: sanitize all user input, use `textContent` instead of `innerHTML`, implement Content Security Policy headers, use framework auto-escaping (React, Angular, Vue all auto-escape by default). **CSRF (Cross-Site Request Forgery)** — tricking authenticated users into making unwanted requests. Prevention: CSRF tokens, `SameSite` cookie attribute. **Content Security Policy (CSP)** — HTTP header controlling which scripts, styles, and resources a page can load. **Supply chain attacks** are the #1 JavaScript security concern in 2025 — the Polyfill.io breach in 2024 compromised 100,000+ websites. Mitigations: audit dependencies with `npm audit`, pin versions, use lockfiles, verify package integrity. **Prototype pollution** — manipulating `Object.prototype` to affect all objects. Avoid recursive cloning without validation. **CORS** — understand it as a security mechanism, not a bug. Always validate on the server — client-side validation is UX only. Serve everything over HTTPS. Never embed secrets in client-side code.

**Web Components.** **Custom elements**: `class MyElement extends HTMLElement { connectedCallback() { } }` registered with `customElements.define('my-element', MyElement)`. Lifecycle callbacks: `connectedCallback`, `disconnectedCallback`, `attributeChangedCallback`, `adoptedCallback`. **Shadow DOM**: `this.attachShadow({ mode: 'open' })` creates an encapsulated DOM tree with isolated styles. Slots (`<slot>`) for content projection. Shadow DOM styling: `:host`, `:host()`, `::slotted()`. **HTML templates**: `<template>` elements hold inert HTML that can be cloned and inserted. **Lit** — the leading lightweight library for building Web Components (by Google), using tagged template literals for reactive templating.

**Web Workers and advanced browser APIs.** **Web Workers** run JavaScript in background threads — no DOM access, communicate via `postMessage()`/`onmessage`. **Service Workers** intercept network requests, enabling offline-first PWAs, caching strategies (`Cache API`), push notifications, and background sync. **SharedArrayBuffer** and **Atomics** for shared memory concurrency. **Streams API** (ReadableStream, WritableStream, TransformStream) for processing data incrementally.

**Algorithms and data structures** (for completeness). Understand Big O notation. Implement in JavaScript: linked lists, stacks, queues, hash tables, binary trees, graphs. Know sorting algorithms (merge sort, quicksort) and searching algorithms (binary search). These are not JavaScript-specific but essential for interviews and efficient problem-solving.

## What mastery actually looks like

After completing all 10 phases, you will understand JavaScript at the level where you can read the ECMAScript specification and make sense of it, debug any JavaScript issue by reasoning about the event loop, prototype chain, and execution context, write performant code that avoids memory leaks and unnecessary reflows, build production applications with proper testing, bundling, and security practices, evaluate new frameworks by understanding what they abstract away, and contribute to open-source JavaScript tools and libraries.

**Three principles for the journey.** First, **build constantly** — every phase should include at least one non-trivial project. Tutorials create the illusion of understanding; building forces you to confront real gaps. Second, **read source code** — once you're past Phase 6, start reading the source of small open-source libraries (Lodash utilities, Zustand, Preact) to see how experienced developers apply these concepts. Third, **teach what you learn** — explaining closures, the event loop, or prototypes to someone else (or in writing) is the fastest way to discover what you don't actually understand.

JavaScript is not a language you "finish" learning — the TC39 committee ships new features annually, frameworks evolve, and best practices shift. But after completing this roadmap, you will have the deep foundational understanding that makes adapting to any change effortless. New features will be variations on patterns you already know, not mysteries. That is what mastery means.
